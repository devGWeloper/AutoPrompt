# Plan: PM 신규 테이블 Oracle DDL (직접 생성용 SQL)

> ⚠️ **SUPERSEDED (2026-05-27).** RAGAS 중심 전환 후 PM 소유 테이블은 **10개 → 6개**로 줄었다
> (`PM_TEST_RUN`/`PM_TEST_RESULT`/`PM_FLOW_VER`/`PM_FLOW_VER_NODE` 삭제, `alembic 0008`).
> 또한 `backend/sql/ddl_initial.sql` 은 더 이상 "폐기 대상"이 아니라 **현행 권위 DDL(6개 테이블)**로
> 갱신됐다. 아래 10개 테이블 DDL은 과거 계획 기록이며, 현행 스키마는
> `backend/sql/ddl_initial.sql` + `backend/app/models/*` 를 본다.

## Context
사용자가 공유 Oracle DB에 **PM(프롬프트 관리) 시스템이 새로 만들어야 하는 테이블**을 코드(ORM create_all)가 아니라
**직접 SQL DDL**로 생성하려 한다.

- 기존 `backend/sql/ddl_initial.sql` 은 **구버전 스키마**(PM_PROJECT/PM_NODE/PM_NODE_EDGE/PM_MODEL_CONFIG/
  PM_PROMPT_VERSION…)로, 현재 ORM 모델과 **불일치**(폐기 대상).
- 현재 구조(권위 기준 = `app/models/*` + `scripts/create_pm_tables.py`):
  - **PM 소유(새로 생성, 10개)**: `PM_NODE_PROMPT_VER`, `PM_FLOW_VER`, `PM_FLOW_VER_NODE`,
    `PM_TEST_DATASET`, `PM_TEST_CASE`, `PM_TEST_RUN`, `PM_TEST_RESULT`, `PM_RAGAS_RUN`,
    `PM_RAGAS_RESULT`, `PM_AUDIT_LOG`.
  - **에이전트 소유(생성 금지, 이미 존재 / FK로 참조만)**: `CHAT_VER_MAS`, `NODE_MAS`, `MODEL_MAS`.

목표: 현재 모델과 1:1로 일치하는 Oracle DDL을 `backend/sql/ddl_pm.sql` 로 제공(직접 실행 가능).

## 매핑 규칙 (SQLAlchemy → Oracle)
Integer→`NUMBER`, String(n)→`VARCHAR2(n)`, Text→`CLOB`, Numeric(p,s)→`NUMBER(p,s)`,
DateTime→`TIMESTAMP`, `Identity(always=True)`→`GENERATED ALWAYS AS IDENTITY`,
`server_default=current_timestamp`→`DEFAULT SYSTIMESTAMP`, String(1) 플래그→`VARCHAR2(1)`.
FK는 모델에 선언된 것만 생성 — `PM_FLOW_VER.CHAT_VER_ID`, `PM_FLOW_VER_NODE.NODE_MAS_ID`,
`PM_TEST_RUN.AB_GROUP_ID` 는 모델상 FK 없음(일반 NUMBER).

## 생성 순서 (FK 의존성)
PM_NODE_PROMPT_VER → PM_FLOW_VER → PM_FLOW_VER_NODE → PM_TEST_DATASET → PM_TEST_CASE →
PM_TEST_RUN → PM_TEST_RESULT → PM_RAGAS_RUN → PM_RAGAS_RESULT → PM_AUDIT_LOG.
(전제: `CHAT_VER_MAS`, `NODE_MAS` 가 먼저 존재해야 FK 생성 가능.)

## Deliverable: `backend/sql/ddl_pm.sql`

```sql
-- ============================================================
-- Prompt Management (PM) — PM-owned tables DDL  (Oracle 12c+/19c/XE21c)
-- 에이전트 소유 CHAT_VER_MAS / NODE_MAS / MODEL_MAS 는 생성하지 않음(이미 존재, FK 참조만).
-- 실행 전 위 외부 테이블이 존재해야 함.
-- ============================================================

-- 1) 노드별 프롬프트 버전 (PM 소유). NODE_MAS.ID 참조 + 자기참조(PREV_PROMPT_ID).
CREATE TABLE PM_NODE_PROMPT_VER (
    PROMPT_ID       NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    NODE_MAS_ID     NUMBER NOT NULL REFERENCES NODE_MAS(ID),
    NODE_NM         VARCHAR2(200) NOT NULL,
    VERSION_NO      VARCHAR2(20) NOT NULL,
    SYSTEM_PROMPT   CLOB,
    USER_PROMPT     CLOB,
    MODEL_NM        VARCHAR2(100),
    TEMPERATURE     NUMBER(3,2),
    MAX_TOKENS      NUMBER,
    TOP_P           NUMBER(3,2),
    EXTRA_PARAMS    CLOB,
    IS_ACTIVE       VARCHAR2(1) DEFAULT 'N',
    CHANGE_SUMMARY  VARCHAR2(500),
    CHANGE_REASON   VARCHAR2(1000),
    PREV_PROMPT_ID  NUMBER REFERENCES PM_NODE_PROMPT_VER(PROMPT_ID),
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP,
    UPDATED_DT      TIMESTAMP,
    CONSTRAINT UQ_PM_NODE_PROMPT_VER UNIQUE (NODE_MAS_ID, VERSION_NO)
);

-- 2) 전체(플로우) 버전 스냅샷. CHAT_VER_ID 는 모델상 FK 없음(일반 NUMBER).
CREATE TABLE PM_FLOW_VER (
    FLOW_VER_ID     NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    CHAT_VER_ID     NUMBER NOT NULL,
    FLOW_VERSION_NO VARCHAR2(20) NOT NULL,
    GRAPH_STRUCT    CLOB,
    MAIN_MODEL_NM   VARCHAR2(100),
    IS_ACTIVE       VARCHAR2(1) DEFAULT 'N',
    CHANGE_SUMMARY  VARCHAR2(500),
    CHANGE_REASON   VARCHAR2(1000),
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 3) 전체 버전별 노드 매니페스트. NODE_MAS_ID 는 모델상 FK 없음.
CREATE TABLE PM_FLOW_VER_NODE (
    ID              NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    FLOW_VER_ID     NUMBER NOT NULL REFERENCES PM_FLOW_VER(FLOW_VER_ID),
    NODE_MAS_ID     NUMBER NOT NULL,
    NODE_NM         VARCHAR2(200) NOT NULL,
    PROMPT_ID       NUMBER REFERENCES PM_NODE_PROMPT_VER(PROMPT_ID),
    VERSION_NO      VARCHAR2(20)
);

-- 4) 테스트 데이터셋. NODE-scope 는 NODE_MAS 참조, FLOW-scope 는 NULL.
CREATE TABLE PM_TEST_DATASET (
    DATASET_ID      NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    NODE_MAS_ID     NUMBER REFERENCES NODE_MAS(ID),
    SCOPE           VARCHAR2(10) DEFAULT 'NODE',
    DATASET_NM      VARCHAR2(200) NOT NULL,
    DESCRIPTION     VARCHAR2(500),
    IS_ACTIVE       VARCHAR2(1) DEFAULT 'Y',
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 5) 테스트 케이스.
CREATE TABLE PM_TEST_CASE (
    CASE_ID         NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    DATASET_ID      NUMBER NOT NULL REFERENCES PM_TEST_DATASET(DATASET_ID),
    INPUT_DATA      CLOB NOT NULL,
    EXPECTED_OUTPUT CLOB,
    EVAL_CRITERIA   CLOB,
    CASE_TYPE       VARCHAR2(50) DEFAULT 'NORMAL',
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 6) 테스트 실행. AB_GROUP_ID 는 모델상 FK 없음.
CREATE TABLE PM_TEST_RUN (
    RUN_ID          NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    RUN_TYPE        VARCHAR2(20) NOT NULL,
    NODE_MAS_ID     NUMBER REFERENCES NODE_MAS(ID),
    CHAT_VER_ID     NUMBER REFERENCES CHAT_VER_MAS(ID),
    PROMPT_ID       NUMBER REFERENCES PM_NODE_PROMPT_VER(PROMPT_ID),
    DATASET_ID      NUMBER REFERENCES PM_TEST_DATASET(DATASET_ID),
    AB_GROUP_ID     NUMBER,
    STATUS          VARCHAR2(20) DEFAULT 'PENDING',
    TOTAL_CASES     NUMBER DEFAULT 0,
    PASSED_CASES    NUMBER DEFAULT 0,
    FAILED_CASES    NUMBER DEFAULT 0,
    AVG_LATENCY_MS  NUMBER,
    TOTAL_TOKENS    NUMBER,
    STARTED_DT      TIMESTAMP,
    ENDED_DT        TIMESTAMP,
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 7) 케이스별 테스트 결과.
CREATE TABLE PM_TEST_RESULT (
    RESULT_ID       NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    RUN_ID          NUMBER NOT NULL REFERENCES PM_TEST_RUN(RUN_ID),
    CASE_ID         NUMBER REFERENCES PM_TEST_CASE(CASE_ID),
    ACTUAL_OUTPUT   CLOB,
    IS_PASSED       VARCHAR2(1),
    EVAL_DETAIL     CLOB,
    LATENCY_MS      NUMBER,
    INPUT_TOKENS    NUMBER,
    OUTPUT_TOKENS   NUMBER,
    ERROR_MSG       VARCHAR2(1000),
    EXECUTED_DT     TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 8) RAGAS 실행(집계). node/prompt/chat 은 FLOW-scope 에서 NULL.
CREATE TABLE PM_RAGAS_RUN (
    RAGAS_RUN_ID       NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    NODE_MAS_ID        NUMBER REFERENCES NODE_MAS(ID),
    PROMPT_ID          NUMBER REFERENCES PM_NODE_PROMPT_VER(PROMPT_ID),
    CHAT_VER_ID        NUMBER REFERENCES CHAT_VER_MAS(ID),
    DATASET_ID         NUMBER NOT NULL REFERENCES PM_TEST_DATASET(DATASET_ID),
    STATUS             VARCHAR2(20) DEFAULT 'PENDING',
    FAITHFULNESS       NUMBER(5,4),
    ANSWER_RELEVANCY   NUMBER(5,4),
    CONTEXT_PRECISION  NUMBER(5,4),
    CONTEXT_RECALL     NUMBER(5,4),
    ANSWER_CORRECTNESS NUMBER(5,4),
    JUDGE_PROVIDER     VARCHAR2(50),
    JUDGE_MODEL        VARCHAR2(100),
    METRICS            CLOB,
    ENGINE             VARCHAR2(20),
    ERROR_MSG          CLOB,
    STARTED_DT         TIMESTAMP,
    ENDED_DT           TIMESTAMP,
    CREATED_BY         VARCHAR2(50) NOT NULL,
    CREATED_DT         TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 9) RAGAS 케이스별 결과.
CREATE TABLE PM_RAGAS_RESULT (
    RAGAS_RESULT_ID    NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    RAGAS_RUN_ID       NUMBER NOT NULL REFERENCES PM_RAGAS_RUN(RAGAS_RUN_ID),
    CASE_ID            NUMBER REFERENCES PM_TEST_CASE(CASE_ID),
    QUESTION           CLOB,
    ANSWER             CLOB,
    CONTEXTS           CLOB,
    GROUND_TRUTH       CLOB,
    FAITHFULNESS       NUMBER(5,4),
    ANSWER_RELEVANCY   NUMBER(5,4),
    CONTEXT_PRECISION  NUMBER(5,4),
    CONTEXT_RECALL     NUMBER(5,4),
    ANSWER_CORRECTNESS NUMBER(5,4),
    ERROR_MSG          CLOB,
    CREATED_DT         TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 10) 감사 로그.
CREATE TABLE PM_AUDIT_LOG (
    LOG_ID          NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    TARGET_TABLE    VARCHAR2(50) NOT NULL,
    TARGET_ID       NUMBER NOT NULL,
    ACTION          VARCHAR2(20) NOT NULL,
    BEFORE_VALUE    CLOB,
    AFTER_VALUE     CLOB,
    CREATED_BY      VARCHAR2(50) NOT NULL,
    CREATED_DT      TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- (선택) 권장 인덱스 — ORM엔 없지만 조회 성능용. 에이전트는 NODE_NM 으로 활성 프롬프트를 읽음.
CREATE INDEX IDX_PM_PROMPT_NODENM_ACT ON PM_NODE_PROMPT_VER (NODE_NM, IS_ACTIVE);
CREATE INDEX IDX_PM_PROMPT_NODE      ON PM_NODE_PROMPT_VER (NODE_MAS_ID);
CREATE INDEX IDX_PM_FLOWVER_CHAT_ACT ON PM_FLOW_VER (CHAT_VER_ID, IS_ACTIVE);
CREATE INDEX IDX_PM_FLOWVERNODE_FV   ON PM_FLOW_VER_NODE (FLOW_VER_ID);
CREATE INDEX IDX_PM_TESTRUN_AB       ON PM_TEST_RUN (AB_GROUP_ID);
CREATE INDEX IDX_PM_TESTRESULT_RUN   ON PM_TEST_RESULT (RUN_ID);
CREATE INDEX IDX_PM_RAGASRESULT_RUN  ON PM_RAGAS_RESULT (RAGAS_RUN_ID);
CREATE INDEX IDX_PM_AUDIT_TARGET     ON PM_AUDIT_LOG (TARGET_TABLE, TARGET_ID);
```

## (선택) 부록: 에이전트 소유 외부 테이블 DDL — 빈/개발 DB에서만
> 운영 공유 DB엔 **이미 존재**하므로 생성하지 말 것. 로컬 개발용 빈 DB에서 PM 테이블 FK를 만족시키려는
> 경우에만 사용. PK는 `GENERATED BY DEFAULT AS IDENTITY`(시드가 명시 ID 삽입 가능).

```sql
CREATE TABLE CHAT_VER_MAS (
    ID            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    GRAPH_STRUCT  CLOB,
    MAIN_MODEL_NM VARCHAR2(100),
    CREATE_DATE   TIMESTAMP DEFAULT SYSTIMESTAMP,
    UPDATE_DATE   TIMESTAMP,
    CREATE_USER   VARCHAR2(50),
    UPDATE_USER   VARCHAR2(50)
);
CREATE TABLE NODE_MAS (
    ID                        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    CHAT_VER_ID               NUMBER NOT NULL REFERENCES CHAT_VER_MAS(ID),
    NODE_NM                   VARCHAR2(200) NOT NULL,
    MODEL_NM                  VARCHAR2(100),
    NODE_DESC                 VARCHAR2(1000),
    PROMPT                    CLOB,
    PROMPT_EDIT_ENABLE_YN     VARCHAR2(1) DEFAULT 'N',
    MODEL_EDIT_ENABLE_YN      VARCHAR2(1) DEFAULT 'N',
    MAIN_MODEL_EDIT_ENABLE_YN VARCHAR2(1) DEFAULT 'N',
    CREATE_DATE               TIMESTAMP DEFAULT SYSTIMESTAMP,
    UPDATE_DATE               TIMESTAMP,
    CREATE_USER               VARCHAR2(50),
    UPDATE_USER               VARCHAR2(50)
);
CREATE TABLE MODEL_MAS (
    ID            NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    GAIA_MODEL_NM VARCHAR2(100) NOT NULL
);
```

## 구현 단계
1. 위 DDL을 `backend/sql/ddl_pm.sql` 로 작성(외부 테이블은 끝에 주석 섹션으로 포함).
2. 구버전 `backend/sql/ddl_initial.sql` 은 현재 코드와 불일치 → 헤더에 "DEPRECATED" 주석을 달거나
   삭제(사용자 확인 후). 기본은 ddl_pm.sql 신규 추가만.

## Verification
- 모델 ↔ DDL 자동 대조: `backend/.venv/Scripts/python.exe backend/scripts/create_pm_tables.py --dry-run`
  → 생성 대상 PM_* 목록이 위 10개와 일치하는지 확인(외부 3개 제외 확인).
- (Oracle 접속 가능 시) `sqlplus <user>/<pw>@<dsn> @backend/sql/ddl_pm.sql` 로 실행 후
  `SELECT table_name FROM user_tables WHERE table_name LIKE 'PM\_%' ESCAPE '\';` 로 10개 생성 확인.
- 회귀: SQLite 테스트는 모델로 생성되므로 `pytest -q` 통과로 스키마 정합성 간접 확인.

## Notes
- `CHAT_VER_MAS / NODE_MAS / MODEL_MAS` 는 절대 생성/변경하지 않음(에이전트 소유). PM은 FK 참조 + NODE_MAS 의
  PROMPT/MODEL_NM/UPDATE_DATE 만 활성화 시 갱신.
- 재실행(드롭) 스크립트가 필요하면 별도 요청 시 `DROP TABLE … CASCADE CONSTRAINTS` 역순 블록 추가 가능.
- `DEFAULT SYSTIMESTAMP` 는 ORM의 `CURRENT_TIMESTAMP` 와 기능 동등(원하면 CURRENT_TIMESTAMP로 교체 가능).
